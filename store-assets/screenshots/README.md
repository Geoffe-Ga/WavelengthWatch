# Screenshot Capture Plan

Apple requires 3–10 screenshots per watch size. This document captures
**what** to shoot, **where** to shoot it, and **how** to export. Final
PNGs are committed alongside this file once captured.

> **Status**: Plan only. No images committed yet — capture on a real device
> (or simulator) before App Store submission.

## 1. Required sizes

App Store Connect groups Apple Watch screenshots by bezel height. For v1.0,
supply the three screenshot groups used by currently shipping watches:

| Group | Resolution | Devices |
|-------|------------|---------|
| 40 mm | 368 × 448 | Apple Watch Series 4–6 (40mm), SE (40mm) |
| 44 mm | 396 × 484 | Apple Watch Series 4–6 (44mm), SE (44mm), Series 7–10 (41mm) |
| 45 / 49 mm | 410 × 502 | Apple Watch Series 7–10 (45mm), Ultra, Ultra 2 |

Optional but recommended: also supply Ultra-native (410 × 502) to make sure
the image isn't cropped.

If you only ship one hardware group at launch, Apple will upscale/downscale
it for display; all three are still preferable to avoid letterboxing.

## 2. Screens to capture

Exactly these five screens, in this order, so the store listing reads as a
narrative:

1. **Privacy-first welcome** — show the privacy toggle or the "local only"
   indicator. Conveys the product's core promise.
2. **Layer / phase navigator** — the dual-axis view with a visibly rich
   layer selected (e.g., Orange Peaking). Conveys the breadth of the
   curriculum.
3. **Curriculum detail** — medicinal vs. toxic side by side. Conveys the
   Archetypal Wavelength depth.
4. **Strategy picker** — show 2–3 strategies for the current phase, with the
   "Log" affordance visible. Conveys one-tap journaling.
5. **Analytics overview** — the on-device analytics home with at least one
   non-empty chart. Conveys the reflection loop.

If there is room for more, add in this priority order:

6. Temporal patterns detail view.
7. Growth indicators detail view.
8. Settings → Cloud Sync, showing the opt-in toggle off.

Keep all five core shots **free of test data**. Seed the device with 10–20
plausible-looking entries across at least three distinct days before
capturing so charts render meaningfully.

## 3. Capture workflow

### On a paired Apple Watch

1. Enable screenshots: **Watch iOS app → General → Enable Screenshots**.
2. On the watch, press **Digital Crown + Side Button** simultaneously. The
   screenshot lands in the iPhone's Photos app.
3. AirDrop to Mac or sync via iCloud Photos.
4. Crop to the target resolution using Preview (⌘K, then enter the exact
   width × height).

### In the simulator

1. Launch the target watch simulator (e.g., `Apple Watch Series 10 (45mm)`).
2. Run the app from Xcode.
3. Seed demo data via the dev tools documented in
   `CLAUDE.md` (reseed backend + enable cloud sync if that screen is being
   captured).
4. Use **File → New Screen Shot** (⌘S) in the Simulator to capture a PNG at
   native resolution.
5. Use `sips -Z <height> input.png --out output.png` or Preview to match the
   exact App Store dimensions.

### Naming convention

Save finished PNGs into this folder using the convention:

```
<order>-<screen>-<group>.png

Examples:
01-privacy-40mm.png
01-privacy-44mm.png
01-privacy-45mm.png
02-navigator-40mm.png
...
```

This makes it easy to diff iterations and to upload in the right order in
App Store Connect.

## 4. Style guidelines

- **No test data.** Seed realistic entries — users can tell when the data
  was generated by a keyboard masher.
- **No PII.** Do not display real user identifiers, emails, or names.
- **No status chrome distractions.** Use the simulator when possible so the
  bezel is clean. If capturing on-device, disable notifications for the
  capture session.
- **Watch face color neutral.** Let the app's Liquid Glass styling carry the
  screenshot; avoid shoots where the bezel blends into the watch face.
- **Locale.** English (U.S.), 12-hour time. Match the locale used for the
  description copy.

## 5. Review checklist

Before uploading to App Store Connect:

- [ ] All 3–10 screenshots present for each of 40mm, 44mm, 45mm groups.
- [ ] Exact pixel dimensions match the target for each group (use
      `sips -g pixelWidth -g pixelHeight <file>` to verify).
- [ ] No PII, no test data visible.
- [ ] Clock, battery, and status icons appear consistent across shots.
- [ ] Order matches the narrative above.
- [ ] Filenames follow the convention in Section 3.

## 6. App preview video (optional)

Apple also accepts a 15–30 second preview video per size group. Scope this
as a follow-on issue — it is not required for first submission.
